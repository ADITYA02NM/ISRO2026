# PS13 Expansion Plans — Beyond the Baseline

> **Scope:** All expansions stay within PS13's four walls (no cloud, no internet at runtime, air-gapped, 4-site MPLS/SD-WAN, offline LLM+RAG, NOC automation).
>
> **Current baseline:** All 4 objectives met, 7 models trained, 3-terminal architecture proven, air-gap validated 39/44.

---

## 🔴 Phase A — "Make It Real" (Close the Loop)

*The system currently detects + explains but does not fix. This phase makes it act autonomously.*

### A1 — Auto-Remediation Engine

When ML ensemble predicts a fault with >95% confidence, automatically execute the recovery before the operator notices.

| Fault Type | Detection Trigger | Auto-Remediation |
|-----------|-------------------|------------------|
| Link Fail | TTI < 1h + XGBoost confidence > 95% | `ip link set <iface> up` via docker exec |
| BGP Flap | Prefix count drop > 30% + anomaly score > 0.8 | `vtysh -c "clear bgp *"` on affected PE |
| Congestion | Bandwidth util > 90% + LSTM forecast rising | Traffic-engineer via BGP community (weighted ECMP) |
| Route Leak | Unexpected prefix in BGP table | Apply prefix-list filter, reset session |
| Node Crash | GNN node_score < 0.2 + no response to ping | Failover to redundant path (OSPF cost manipulation) |
| LSP Break | MPLS tunnel down | Re-signal LDP session, re-establish LSP |
| CRC Errors | CRC error rate > threshold | Interface reset, flap port |

**Implementation:**
```
ml/remediation/
├── engine.py              — Remediation orchestrator (watches predictions, triggers actions)
├── actions/               — One file per recovery action
│   ├── bgp_reset.py      
│   ├── link_bounce.py    
│   ├── traffic_steer.py  
│   ├── prefix_filter.py  
│   └── lsp_resignal.py   
├── safety.py              — Circuit breaker (max 3 auto-fixes per hour, operator override)
└── rollback.py            — Undo actions if recovery makes things worse
```

### A2 — Incident Lifecycle

Turn one-shot predictions into full incident records:

```
Detection → Classification → Auto-Remediation → Verify → Escalate if failed → Post-Incident Report
```

- Store incidents in SQLite at `ml/data/incidents.db`
- Dashboard shows live incident timeline (not just latest prediction)
- Post-incident report generated by LLM: "Fault: BGP Flap on PE1-BLR. Detected at 14:02. Auto-remediated at 14:03. Root cause: Route-map misconfig."

### A3 — Live Telemetry Feed

Replace manual `POST /predict` with automatic streaming pipeline:

```
Containers (5s) → Telegraf → Kafka → ML Consumer → FastAPI WebSocket → Dashboard
```

- Remove the T1 "Trigger Event" button from dashboard — predictions happen automatically every 5s
- Add "Pause Auto-Inference" toggle for demo/training mode
- WebSocket pushes alerts to dashboard without polling

### B2 — Add 6+ Fault Scenarios

Current: 7. Target: 13.

| # | Fault | Script | Detection Strategy |
|---|-------|--------|-------------------|
| 8 | **DNS Poisoning** | `fault-dns-spoof.sh` | DNS response mismatch, TTL anomaly |
| 9 | **DHCP Exhaustion** | `fault-dhcp-exhaust.sh` | Lease pool depletion rate, DORA failure count |
| 10 | **ARP Spoofing** | `fault-arp-spoof.sh` | MAC-IP binding changes, duplicate MACs |
| 11 | **DDoS (slow)** | `fault-ddos-slow.sh` | Connection table growth, SYN flood ratio |
| 12 | **MTU Mismatch** | `fault-mtu-mismatch.sh` | ICMP fragmentation needed, packet drops at MTU |
| 13 | **Certificate Expiry** | `fault-cert-expiry.sh` | remaining_days < 30 on IPsec/SSL certs |

### B3 — Cross-Site Cascade Detection

Train a model that detects **cascading** failures across sites:

```
BLR link fail (t=0) → MUM congestion (t=8min) → CHE latency spike (t=15min)
```

- Uses GNN + temporal attention: graph structure captures inter-site dependencies
- Adds `site_*` cross-correlation features to the 109 existing features
- Dashboard highlights "Cascade Risk" when a single-site fault has propagation potential

### B4 — Incremental / Online Learning

Current model re-training is offline (full retrain). Add:

- **Online XGBoost**: `xgb_model.fit(X_new, y_new, xgb_model=xgb_model)` — incremental update without full retrain
- **Sliding-window retrain**: Autoencoder + LSTM retrained daily on last 7 days of data
- **Drift detection**: If prediction confidence drops below 90% on recent data, trigger retrain

## 🟢 Phase C — "Make It Production" (Air-Gap & Reliability)

*PS13 demands air-gap compliance. Make it audit-proof, demo-proof, and deployable by anyone.*

### C1 — 44/44 Air-Gap Validation

Currently 39/44 pass. Fix the 5 failing checks:

| # | Current Failure | Root Cause | Fix |
|---|----------------|------------|-----|
| 1 | Air-gap DNS check | No interface filter for loopback | Add `--lo` exclusion |
| 2 | Air-gap DNS check (host) | Different DNS resolution path on Arch | Add conditional path |
| 3 | Model ONNX load | ONNX runtime init error | Load only native format where ONNX unavailable |
| 4 | Server health endpoint | Server not running at test time | Pre-condition check |
| 5 | RAG query response | Empty collection at test time | Seed test data if empty |

### C2 — Runtime Air-Gap Guard

Current validation is file-scan only (static). Add runtime enforcement:

- **systemd service**: `airgap-guard.service` that monitors network interfaces and blocks any non-local outbound traffic
- **FastAPI middleware**: Before every request, verify no unexpected egress activity
- **Cron check**: Every 5 minutes, verify DNS resolution returns only local entries (no `google.com` etc.)
- **Dashboard badge**: "🔴 AIR-GAP BREACH" / "🟢 AIR-GAP COMPLIANT" shown at all times

### C3 — One-Command Demo Script

```bash
#!/bin/bash
# ./start-demo.sh — Start everything, open dashboard, begin monitoring

set -e

echo "🔍 Checking prerequisites..."
which containerlab         || { echo "Missing containerlab"; exit 1; }
curl -s http://localhost:11434/api/tags > /dev/null || { echo "Ollama not running"; exit 1; }
test -f ml/venv/bin/python || { echo "venv not found"; exit 1; }

echo "🚀 Deploying Containerlab topology..."
sudo containerlab deploy -t topology.clab.yml

echo "🤖 Starting FastAPI server..."
ml/venv/bin/python -m uvicorn ml.noc_copilot:app --host 0.0.0.0 --port 8000 &
sleep 3

echo "📊 Opening dashboard..."
xdg-open ml/noc-dashboard.html

echo "✅ Demo running. Press Ctrl+C to stop."
wait
```

### D2 — Network Health Score

Single composite score (0–100) combining all 7 model outputs:

| Component | Weight | Source |
|-----------|--------|--------|
| Fault probability | 25% | XGBoost prediction confidence |
| Anomaly severity | 20% | IsolationForest score |
| TTI (time to incident) | 20% | TTI Regressor (inverse: shorter TTI = lower score) |
| Autoencoder reconstruction error | 15% | Higher error = lower score |
| GNN node health (mean) | 10% | Average of all 10 router node_scores |
| LSTM forecast divergence | 5% | How much next-12h diverges from normal |
| Prophet trend slope | 5% | Steeper upward trend = lower score |

Displayed on dashboard as a large gauge / speedometer.
