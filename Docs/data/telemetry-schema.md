# Telemetry Schema — `ml/data/telemetry.parquet`

**7,920 rows × 109 columns.** Synthetic network telemetry data simulating 11 devices across 4 sites at hourly intervals (7920 = 11 × 4 × 30 × 6 = ~6 months × 4 sites × 11 devices × ~30 days).

## Core Features (12)

| Feature | Type | Description |
|---------|------|-------------|
| `latency` | float (ms) | Round-trip latency |
| `jitter` | float (ms) | Latency variation |
| `packet_loss` | float (%) | Packet loss percentage |
| `throughput` | float (Mbps) | Data throughput |
| `cpu_usage` | float (%) | Device CPU utilization |
| `memory_usage` | float (%) | Device memory utilization |
| `temperature` | float (°C) | Device temperature |
| `link_utilization` | float (%) | Link bandwidth utilization |
| `ospf_neighbors` | int | OSPF adjacency count |
| `bgp_prefixes` | int | BGP prefix count |
| `mpls_lsp_count` | int | MPLS LSP count |
| `discards` | int | Packet discards |

## Context Fields (4)

| Field | Type | Description |
|-------|------|-------------|
| `device_id` | str | Unique device identifier (e.g., BLR-CE-1, MUM-PE-2) |
| `site` | str | Site code: BLR, MUM, CHE, DEL |
| `device_role` | str | Role: CE (Customer Edge), PE (Provider Edge), P (Provider) |
| `timestamp` | datetime | Hourly timestamps |

## Engineered Features (93)

Generated by [[ensemble-predictor]] at inference time:

| Category | Count | Pattern |
|----------|-------|---------|
| Lags (1h, 2h, 3h, 6h, 12h) | 60 | `{feature}_lag_{N}h` |
| Rolling means (3h, 6h, 12h) | 36 | `{feature}_rolling_mean_{N}h` |
| Rolling stds (3h, 6h, 12h) | — | Included in rolling_mean set |
| Site means | 12 | `{feature}_site_mean` |
| Cycle encoding | 2 | `hour_sin`, `hour_cos` |
| Passthrough | 16 | 12 core + 4 context |

## Construction

```bash
ml/venv/bin/python ml/data/generate_synthetic_data.py
```

Generates realistic telemetry with injected fault patterns (BGP flap → latency spikes + prefix drops, congestion → throughput cap + jitter, etc.)
