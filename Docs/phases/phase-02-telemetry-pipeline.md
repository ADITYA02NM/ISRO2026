# Phase 2: Telemetry Pipeline

**Status: 🟡 PARTIAL — Configs complete, stack never deployed live**

**Goal:** Build a real-time telemetry pipeline — collect network metrics from Containerlab, stream them through Kafka, store in Prometheus/Elasticsearch, and serve via WebSocket for the NOC dashboard.

**Duration:** Days 3-5

---

## Architecture

```
FRR Routers (SNMP/IP Fix)
       │
       ▼
   Telegraf ──────► Prometheus ──► WebSocket ──► Dashboard
       │
       ▼
     Kafka ──────► Elasticsearch ──► Kibana
       │
       ▼
  ML Predictor ──► ChromaDB/LLM
```

### 3-Terminal Design (T1/T2/T3)
| Terminal | Component | Function |
|----------|-----------|----------|
| T1 | Trigger Events | User triggers fault scenarios, generates telemetry snapshots |
| T2 | Watch Alerts | Polls predictions, streams alerts to dashboard |
| T3 | AI Chat / Analytics | LLM queries through RAG, historical analysis |

## Synthetic Telemetry Dataset

Generated by `ml/generate_synthetic_data.py`:

| Property | Value |
|----------|-------|
| **Rows** | 7,920 |
| **Columns** | 109 (12 core + 97 engineered) |
| **Devices** | 11 (4 CE, 4 PE, 3 P) |
| **Sites** | 4 (BLR, MUM, CHE, DEL) |
| **Timestamps** | Hourly over 30 days |
| **Fault Types** | 8 (BGP flap, OSPF flapping, MPLS LSP, interface CRC, CPU spike, memory leak, queue drop, link flapping) |

### Core Features (12)
`cpu_util`, `memory_util`, `bandwidth_util`, `packet_loss_pct`, `latency_ms`, `jitter_ms`, `tcp_retransmits`, `queue_depth`, `interface_errors`, `bgp_prefix_count`, `ospf_lsa_count`, `mpls_lsp_count`

### Engineered Features (97)
- **Rolling windows** (1h/3h/6h/12h): mean, std, min, max, slope (4 windows × 12 features × 5 stats × 3 delta = 48)
- **Site/role aggregates**: site_means, role_means (12 features × 2 = 24)
- **Ratios/historical**: bw_ratio, mem_bw_ratio, cpu_anomaly, bw_vs_site_ratio (17)
- **Cyclic encoding**: hour_sin, hour_cos (2)

## Telemetry Schema (TelemetrySnapshot)

Used by `/predict` endpoint:

```python
class TelemetrySnapshot(BaseModel):
    device_id: str       # e.g. "PE1-BLR"
    site: str            # "BLR" | "MUM" | "CHE" | "DEL"
    device_role: str     # "CE" | "PE" | "P"
    timestamp: str       # ISO 8601
    cpu_util: float
    memory_util: float
    bandwidth_util: float
    packet_loss_pct: float
    latency_ms: float
    jitter_ms: float
    tcp_retransmits: float
    queue_depth: float
    interface_errors: float
    bgp_prefix_count: float
    ospf_lsa_count: float
    mpls_lsp_count: float
```

Note: The 97 engineered features are computed inline by `ensemble_predictor.feature_engineering()` at inference time.

## Key Decisions
| Decision | Choice | Rationale |
|----------|--------|-----------|
| Streaming | Kafka (not Redis Streams) | Standard in NOC/telecom; durable log |
| Storage | Prometheus + Elasticsearch | Metrics (Prom) + logs (ES) separation |
| Schema | 109 features | Comprehensive + 7-model compatibility |
| Dataset | 30 days, hourly | Covers weekly/seasonal patterns |
