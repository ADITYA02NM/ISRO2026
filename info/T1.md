# T1 — Operator Console

> **Role:** You, the operator, trigger events on devices from here
> **Port:** 5173
> **Target:** Frontend for commanding device actions (ping, traceroute, anomaly injection, lockdown, etc.)
> **Stack:** React 18 + Vite + Three.js (R3F) + anime.js + Tailwind
> **Input Mode:** Paste this prompt into a code-generation tool

---

## Stitch Prompt

```
Make me a jaw-dropping, space-themed operator console using React, Three.js (via @react-three/fiber + @react-three/drei), anime.js, and Tailwind CSS. This is Terminal 1 of the PS13 Mission Control system — an air-gapped predictive copilot for secure MPLS ground-station operations.

This is NOT just a dashboard. This is a COMMAND CONSOLE. You look at the live network state, then trigger actions on devices.

### 🧭 Vibe & Theme
Full-space-ops aesthetic. Deep starfield background (parallax layered). Glowing cyan/teal accent lines (#00f0ff, #0a4d8c, #7b2ffc). Subtle scan-line overlay on panels. Every element feels like a NASA JPL or ISRO mission control terminal. The UI should look alive at all times — data pulses, orbit trails, soft breathing glows on every interactive element.

### 🗺️ Core Scene — India Map with Network Topology
Render a stylized wireframe map of India using Three.js. Use GeoJSON data for India's outline — draw it as glowing line segments. Place prominent glowing network nodes at these 4 cities with their ISRO/ground-station names:
- **Bangalore** (ISRO HQ — NETWORK OPS) → cyan glow, largest node
- **Mumbai** (WEST GATEWAY) → teal glow
- **Chennai** (EAST GATEWAY) → blue glow
- **Delhi** (NORTH RELAY) → purple glow

Each node must have:
- Pulsating outer ring expanding outward
- Orbiting micro-satellites (tiny glowing spheres in elliptical orbits around the node)
- Online/offline status indicator (green/red dot)
- A subtle label in a modern tech font (Orbitron or Rajdhani)

Node status comes LIVE from T3 over WebSocket — do NOT hardcode it.

### 📡 Data Connection to T3
- WebSocket: `ws://localhost:5174/ws/topology` — receives live device state, node health, link status
- REST: `POST http://localhost:5174/api/device/:id/:action` — fires commands
- The connection status indicator must show green (connected), amber (reconnecting), or red (disconnected)

### 🎮 Device Actions — THE CORE FEATURE
Click any city node on the map. The camera smoothly zooms in (anime.js tween), and an action panel slides in from the right with these commands:

| Action | Endpoint | Description |
|--------|----------|-------------|
| **Ping** | `POST /api/device/:id/ping` | Send ICMP ping to peer device |
| **Traceroute** | `POST /api/device/:id/traceroute` | Trace route to peer |
| **Bounce Interface** | `POST /api/device/:id/bounce` | Reset device interface |
| **Trigger Anomaly** | `POST /api/device/:id/anomaly` | Inject unusual behavior (testing) |
| **Lockdown Route** | `POST /api/device/:id/lockdown` | Lockdown all routes on device |
| **Diagnostic Mode** | `POST /api/device/:id/diagnostic` | Toggle diagnostic mode on/off |

Each action button:
- Shows a loading spinner while T3 processes
- Turns green on success, red on failure
- Displays the response message from T3
- Logs the action in the Command History panel

### 📦 Command History Panel
A collapsible panel (bottom-left or slide-in from left) showing a scrollable log of every command you've fired:

```
[14:32:47] PING → Bangalore → SUCCESS (rtt: 4.2ms)
[14:33:12] ANOMALY → Mumbai → TRIGGERED
[14:33:45] LOCKDOWN → Delhi → ACTIVE
[14:34:01] DIAGNOSTIC → Chennai → RUNNING
```

- Status icons: ⏳ Pending → ✅ Success → ❌ Failed → 🔄 Running
- Timestamp + device + action + result
- Click any entry to re-fire the command
- Clear log button

### 📡 Packet Transfer Animations (anime.js)
Data packets flow between cities along curved arc paths. These reflect REAL command traffic:
- When YOU fire a Ping, a cyan command packet animates from source to target
- When YOU trigger an anomaly, amber anomaly packets pulse from your node
- When T3 sends back a result, green ack packets return
- Multiple simultaneous transfers keep the scene alive
- Trail effect — fading motion trail behind each packet
- Color-coded: commands (cyan), anomalies (amber), acknowledgements (green)

### 🔍 Zoom-to-Device Interaction
When user clicks a city node, smoothly animate the Three.js camera to zoom into that node. Show:
- Device detail panel with live metrics from T3 (latency, throughput, uptime, active routes)
- The action buttons panel (described above)
- A "Back to Overview" button that reverse-zooms the camera out
- Use drei's `CameraControls` or custom `useFrame` tweening with anime.js

### 🎛️ Hover Panel with Toggles
On the right side, a collapsible control panel with:
- **Node Visibility**: toggle city node visibility (4 checkboxes)
- **Packet Animation**: toggle on/off packet animations, adjust speed slider
- **Theme Switcher**: toggle between Space (dark) and Aurora (deep blue-purple)
- **Auto-Rotate**: toggle camera auto-orbit around the map
- **Command History**: toggle show/hide the command log panel
- Each toggle uses the `theme-toggle-buttons` style — animated slide-knob with glow
- Sections collapsible with smooth height animation (anime.js)

### 📊 HUD Overlay
Persistent heads-up display across the top and corners:
- **Top bar**: "PS13 MISSION CONTROL • TERMINAL 1" with blinking red recording dot and live timestamp
- **Top-right**: "T3 LINK: CONNECTED" with signal strength bars (4 animated bars in cyan)
- **Bottom-left**: "DEVICES: 4/4 ONLINE" — live from WS, "COMMANDS FIRED: 12" counter
- **Bottom-right**: system heartbeat pulse animation, WebSocket ping/pong indicator
- All HUD text in Orbitron/Space Mono, semi-transparent with subtle glow

### 🧩 Reference Components to Incorporate
Use these existing components from the project's component library (treat as ready-made imports — do NOT recreate them):

1. **WorldMap** (from `map.tsx`) — integrate as the base Three.js map layer, overlay network nodes on top
2. **radial-orbital-timeline** — use this component's orbital ring logic for the node satellite orbits
3. **flightcn-satellite-orbits** — adapt the satellite trail rendering for packet paths
4. **theme-toggle-buttons** — use for all toggle switches in the hover panel
5. **container-scroll-animation** — wrap the action panel / command history in this for scroll-reveal effects
6. **system-monitor** — embed this as a mini-widget showing aggregate device health
7. **animated-hud-targeting-ui** — use the crosshair/reticle logic for the node targeting cursor on hover
8. **processing-card** — use for the action button cards and device detail panel

### 🌟 Extra Cool Factor
- Add a subtle constellation particle system in the background (stars that subtly connect into random lines)
- Each city node emits a soft light cone upward (like a searchlight)
- When a command succeeds, emit a tiny green ring ripple at the target node
- Keyboard shortcuts: press `H` to toggle HUD, `M` to toggle map labels, `Space` to pause/resume packets, `C` to clear command history
- Command confirmation dialog for destructive actions (Lockdown, Bounce)

### 📱 Responsive
- Desktop-first but the canvas and HUD reflow gracefully at tablet sizes
- The action panel becomes a bottom drawer on mobile
- Command history becomes a full-screen overlay on mobile
- Map auto-adapts to viewport

Make it feel like a REAL mission control terminal — every pixel should scream SPACE OPS and COMMAND AUTHORITY.
```

## Usage

1. Copy the entire prompt block above
2. Paste into the code-gen tool with target directory `src/terminals/terminal1/`
3. After generation, install deps: `npm install three @react-three/fiber @react-three/drei animejs`
4. Verify all referenced components exist in the component library before the tool runs
