# T3 — Express + React Frontend Terminal

> **Port:** 3000
> **Role:** SPA server + REST API for NOC dashboard
> **Process:** `npx tsx server.ts`
> **Launcher:** `./ps13-frontend.sh`

---

## Overview

Express server serving three functions:
1. **SPA Host** — Serves built React app (`dist/`) on `GET /`
2. **REST API** — 14 endpoints for NOC data operations
3. **Static Assets** — JS/CSS bundles with cache-busting hashes

## Build Command

```bash
cd /home/ego/Documents/ISRO2026
npx vite build
# Output: dist/ (1.4MB JS + 66KB CSS)
```

## Server Start

```bash
npx tsx server.ts
# Listening on http://localhost:3000
```

## REST API (14 Endpoints)

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | /api/cities | All 4 cities |
| GET | /api/cities/:cityId/devices | Devices per city |
| GET | /api/alerts | Alert feed (pinned + timestamp sorted) |
| GET | /api/alerts/summary | Severity counts |
| GET | /api/devices/:deviceId | Device info |
| GET | /api/devices/:deviceId/health | CPU, memory, uptime |
| GET | /api/system/status | Ollama/FastAPI/Docker probes |
| GET | /api/analytics | Dashboard stats |
| POST | /api/events/trigger | Inject fault |
| POST | /api/events/random-burst | 4 random faults |
| PATCH | /api/alerts/:id/pin | Toggle pin |
| DELETE | /api/alerts/:id | Delete alert |
| POST | /api/chat | Chat via FastAPI proxy |
| POST | /api/dashboard/reset | Full reset |

## In-Memory State

Server maintains state in-memory (no database):
- `cities[]` — 4 city objects with ID, name, devices
- `alerts[]` — mutable alert feed (pin, delete, create)
- `analytics` — computed on GET (aggregated stats)
- `systemStatus` — probed on GET (live checks to Ollama, FastAPI)
